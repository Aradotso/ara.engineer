ae·ara engineer / secrets

Secrets live in Railway.

One source of truth for every runtime credential the team uses — Stripe, OpenAI, Anthropic, Supabase, GitHub, Slack, Resend, Google, Cloudflare, Axiom, Braintrust, and everything else. Agents connected to the Ara MCP fetch them directly. Humans use 1Password.

The canonical store

The Ara Backend project on Railway hosts every Ara service. Its ara-api service carries the bulk of the secrets — anything another service needs, it inherits from there or has its own copy in its own Railway service's variables.

How agents read them

Every MCP client connected to connectors.ara.engineer/mcp gets the Railway tools on session start. The server's instructions tell the agent to skip discovery and go straight to railway_get_variables with the three IDs above. 

{
  "tool": "railway_get_variables",
  "args": {
    "projectId": "5b03413d-9ace-4617-beb5-18b26ce5f339",
    "environmentId": "f3d22dae-9e86-4a38-a26e-0d27efa12749",
    "serviceId": "304984ab-6cc0-42cf-80e4-9cc9b6529b21"
  }
}

Falls back to ara-connectors and other services via railway_list_projects if the key isn't on ara-api.

The rules

1. Never commit secrets.

.env and .env.local are gitignored. If a key leaks to git, rotate it in Railway first, then worry about history.

2. Never roll your own vault.

Railway is the store. 1Password is for humans. Don't build something in between — no custom "ara-secrets" service, no shared sqlite, no "just for dev" helper that drifts into prod.

3. Agents don't ask for keys.

If a cloud agent asks the user to paste a credential, the MCP wiring is broken — the key should already be on a Railway service somewhere. Fix the wiring, don't paste the key.

4. Rotation is a Railway edit.

Update the variable on ara-api, the service redeploys, the new value is live. If something else depends on it, use Railway's shared variables or add the key to that service's own vars.

5. Human secrets stay in 1Password.

Personal Google tokens, SSH keys, and anything tied to a human identity don't go in Railway. Railway is for what services need to run.

Humans: 1Password

Team vault: Ara Engineering. If you need a key that a service runs on, it's in Railway. If you need a login that a human uses, it's in 1Password.
